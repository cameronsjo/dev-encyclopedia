# System Design

Designing scalable, reliable, and maintainable software systems.

## Overview

| Aspect | Details |
|--------|---------|
| Focus | Architecture, scalability, reliability |
| Scope | Distributed systems, databases, caching, networking |
| Key skills | Trade-off analysis, capacity planning |

---

## Core Concepts

### Scalability

| Type | Description |
|------|-------------|
| Vertical | Add more power to existing machines |
| Horizontal | Add more machines |

### Reliability

- Redundancy
- Replication
- Failover mechanisms

### Performance

- Latency vs throughput
- Caching strategies
- Load balancing

---

## Common Patterns

| Pattern | Use Case |
|---------|----------|
| Load Balancer | Distribute traffic |
| CDN | Static content delivery |
| Message Queue | Async processing |
| Database Sharding | Horizontal data scaling |
| Caching | Reduce database load |
